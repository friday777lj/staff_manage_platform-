# 人才服务运营平台：商用开发全景知识库 (Master Blueprint)

## 一、 顶层设计：商业模式与核心逻辑

### 1.1 商业模式：一站式人才服务运营 (Human Capital Operation)
*   **本质：** 本平台不只是“信息发布工具”，而是“人才资产经营中枢”。
*   **核心理念：** 服务产品化。将抽象的校企合作、实习管理转化为标准化的、可度量的数字化产品。
*   **盈利/价值闭环：** 
    *   **对企业：** 构建“人才蓄水池”，降低 50% 以上的招聘与入职培训成本。
    *   **对院校：** 提供“数字化就业指挥系统”，通过过程管理和数据反哺，支撑“双高”建设。
    *   **对学生：** 生成“数字化职业信用绿卡”，让实习经历从“文字描述”变为“数据存证”。

### 1.2 核心业务流：协议驱动 (Agreement-Driven)
**研发核心注意：** 本系统不在线上处理复杂的面试逻辑，而是以“纸质协议上传”作为状态流转的硬性开关。
*   **入岗判定：** “推荐 -> 面试(线下) -> 结果(线下) -> **协议上传(线上触发点)** -> 正式入岗”。

## 二、 业务功能大地图 (优先开发矩阵)

### 2.1 P0 核心闭环 (商用第一优先级)
*   **权限体系：** 多租户隔离 (管理端、学校、企业、学员)。必须确保数据严格物理隔离。
*   **可视化驾驶舱 (看板)：** 
    *   管理员看板：跨区域、跨行业的宏观运营数据。
    *   学校看板：本校实习对口率、在岗状态。
    *   企业看板：人才匹配效率、岗位饥渴度。
*   **关键履约：** 实习协议 PDF/图片上传模块，以及以此触发的状态自动变更引擎。
*   **人岗匹配引擎：** 支持语义搜索和多维度标签匹配。

### 2.2 P1 运营赋能 (第二优先级)
*   **过程管理：** 日报/周报流。
*   **学校权限：** 校内导师“只读”审阅权限；支持老师主动为学生推岗。
*   **双向评价：** 实习结束后的服务质量回传 (学生评企业、企业评学生)。
*   **能力画像：** 整合外部测评数据与内部实训数据，生成数字化人才画像。

## 三、 技术规范建议 (避免“屎山”指南)

### 3.1 架构建议
*   **API First：** 所有的后端逻辑必须通过标准的 RESTful API 暴露，方便未来移动端、小程序、大屏的独立调用。
*   **状态机设计：** 针对“学员状态”和“岗位状态”，必须设计严格的状态机 (State Machine)，防止状态跳跃或逻辑死锁。
*   **异步任务：** 协议上传后的数据处理、反向同步至公共服务平台的数据，建议使用队列辅助。

### 3.2 数据库关键实体 (Entity Relational)
*   **Enterprise (企业)：** 需包含行业分类代码 (需支持国标)。
*   **Job (岗位)：** 需关联“技能地图”标签。
*   **Student (学员)：** 建议采用“基本信息+多维度扩展画像”的方式存储。
*   **Fulfillment (履约)：** 核心表，存储学生、企业的关联关系及协议存证路径。
